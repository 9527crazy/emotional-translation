# 情绪翻译 (Emotional Translation) — 产品需求文档

> v0.2 | 2026-06-22 | Web 端 MVP

---

## 1. 产品定义

**一句话**：通过浏览器摄像头实时捕捉人脸情绪，将抽象情感"翻译"为可理解、可分享的多模态表达。

**为什么做**：情绪是人类最真实的语言，但常常难以言说。本产品搭建情感与表达之间的桥梁。

**为什么 Web 端先发**：
- 零安装，浏览器打开即用，传播成本最低
- 跨平台兼容（桌面 + 移动浏览器）
- 快速迭代，无需应用商店审核
- WebGPU / WebNN 等新技术持续提升端侧推理能力

## 2. 目标用户

| 用户画像 | 典型场景 | 核心痛点 |
|----------|---------|---------|
| 社交困难者 | 社交场合难以读懂他人情绪 | 缺乏情绪识别辅助工具 |
| ASD 群体 | 日常交流中难以理解面部表情 | 需要可视化的社交情绪提示 |
| 内容创作者 | 直播/视频中展示情绪变化 | 需要趣味化的情绪可视化 |
| 心理咨询师 | 来访者情绪状态客观记录 | 需要量化的情绪追踪工具 |
| 普通用户 | 好奇自己的情绪状态 | 想了解自我情绪表达 |

## 3. 核心功能

### 3.1 实时情绪采集
```
浏览器摄像头 → 视频帧捕获 → 人脸检测 → 关键点定位 → 情绪分类 → 实时输出
```

- WebRTC `getUserMedia` 获取摄像头
- 面部 468 关键点定位（MediaPipe Face Mesh）
- 7 种基础情绪：快乐、悲伤、愤怒、惊讶、恐惧、厌恶、中性
- 情绪强度量化（0-100%）
- 光照自适应与遮挡容错

### 3.2 多模态情绪翻译
| 翻译目标 | 输出形式 | 说明 |
|----------|---------|------|
| 色彩翻译 | 情绪→渐变色 | 全屏沉浸式色彩展示 |
| Emoji 翻译 | 情绪→Emoji 动画 | Canvas 浮动动画 |
| 文字翻译 | 情绪→诗意描述 | 模板引擎 / LLM |
| 数据翻译 | 情绪→实时图表 | ECharts 曲线图 |

### 3.3 情绪记录与回顾
- 情绪变化实时曲线
- 日/周情绪趋势统计
- 关键时刻截图标注

### 3.4 分享
- 情绪卡片生成（Canvas 截图 + 文字叠加）
- 图片下载

---

## 5. 隐私原则

- **端侧优先**：情绪识别全部在浏览器完成，不上传面部数据
- **最小数据**：仅存储情绪标签 + 时间戳，不存储面部图像
- **用户可控**：随时清除所有本地数据
- **透明告知**：首次使用清晰说明数据处理方式

## 6. 非功能性需求

| 维度 | 要求 |
|------|------|
| 情绪识别延迟 | < 150ms |
| UI 帧率 | ≥ 24fps |
| 情绪识别准确率 | ≥ 80%（标准光照） |
| 首屏加载 | < 3s（CDN + 代码分割 + 模型懒加载） |
| 离线能力 | 核心功能离线可用（PWA） |
| 无障碍 | 键盘导航 + ARIA + 色彩对比度 ≥ 4.5:1 |

## 8. 竞品差异化

| 竞品 | 特点 | 我们的差异 |
|------|------|-----------|
| Affectiva | B 端情绪分析 | 面向 C 端，强调趣味与艺术性 |
